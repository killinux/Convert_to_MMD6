# 拉直手臂（肘 + 腕）功能说明

> 操作符 `object.straighten_arms` ·  面板按钮「拉直手臂(肘+腕)」 ·  文件 `operators/straighten_arms_operator.py`

## 1. 解决什么问题

很多外部模型（尤其是 **XPS / XNALara**）导入后，手臂在静置姿势(rest pose)下并不是完全伸直的，存在两处预弯：

| 环节 | 含义 | XPS 典型值 |
|------|------|-----------|
| **肘 (ひじ)** | 大臂 → 小臂 的夹角 | ~14° |
| **腕 (手首)** | 小臂 → 手 的夹角 | ~5° |

MMD 的标准动作(VMD)默认假设手臂静置时 `腕 → ひじ → 手首` 基本共线。预弯会让套用动作时肘/腕角度叠加出偏差。本功能把这两处自动检测并拉直成共线。

## 2. 通用性

- 不写死任何骨骼名，从面板的手臂映射属性读取：
  `left/right_upper_arm_bone`、`left/right_lower_arm_bone`、`left/right_hand_bone`。
- 既适用于**重命名前**的原始骨名（如 `arm right elbow`），也适用于**重命名为 MMD** 之后的 `右ひじ` 等。
- 任意一侧/任意环节缺映射都会被安全跳过。

## 3. 核心原理：必须烘焙网格

⚠️ 这是本功能最关键的点，也是手写脚本最容易踩的坑：

> **只把骨骼摆直再 `armature_apply` 是不够的** —— 那样骨骼 rest 变直了，但网格顶点会"弹回"原始弯曲形状，导致**骨骼直、皮肉弯**的错位。

`armature_apply`（应用姿态为静置姿态）只改骨骼，不动网格顶点。正确流程必须**先把当前姿态烘焙进网格顶点**：

```
1. 姿态模式：把子骨旋转到与父骨共线（绕关节头部旋转，关节位置不变）
            顺序：先所有"肘"，再所有"腕"（腕对齐到已拉直的小臂）
2. 烘焙    ：对每个网格 apply 骨架修改器 → 当前形变写入网格顶点
            （带形态键的网格先备份移除，烘焙后再恢复）
3. 固化    ：armature_apply 把直姿态设为新的 rest pose
4. 复原    ：给网格重新挂回骨架修改器（此时形变=单位阵，网格保持笔直）
```

子骨旋转后，其子级（手、手指）作为子级自动跟随，所以整条手臂链一起调整。

## 4. 使用方法

1. 在「Convert to MMD」主面板里映射好手臂骨骼（上臂 / 下臂 / 手），或加载对应预设（如 `xna_lara`）。
2. 选中骨架，点击 **「拉直手臂(肘+腕)」** 按钮。
3. 状态栏会报告检测到的角度与已修正的环节。

### 参数（在操作符重做面板 F9 中可调）

| 参数 | 默认 | 说明 |
|------|------|------|
| `angle_threshold` 阈值(度) | `0.5` | 夹角大于此值才修正；已伸直的模型不会被动 |
| `fix_wrist` 同时拉直手腕 | `True` | 关掉则只拉直肘部，保留手腕自然angle |
| `check_only` 仅检测不修改 | `False` | 干跑：只在状态栏报告各环节角度，不改模型 |

## 5. 测试结果

在 `inase (purifier)_lezisell-A/xps.xps`（109 骨，8 网格，无形态键）上验证：

```
检测(check_only): left 肘 14.38°, left 腕 4.92°, right 肘 14.38°, right 腕 4.92° | 需修正: 4 处
执行修复:         已拉直 4 处: left-肘(14.4°), right-肘(14.4°), left-腕(4.9°), right-腕(4.9°)
修复后:           肘 0.00°, 腕 0.00°（左右两侧）
网格跟随验证:      手腕网格点云 Y = -0.000  ≈  手腕骨 Y = 0.003   （间隙 1.5mm）
```

- 骨骼角度归零 ✅
- 网格顶点跟随骨骼（点云重心贴合骨骼轴，间隙 ~1.5mm）✅
- `check_only` 干跑模式正常 ✅
- 对已伸直的模型重复执行：阈值内跳过，不产生副作用 ✅

## 6. 与「转换为A-Pose」的区别

| | 拉直手臂 | 转换为A-Pose |
|---|---|---|
| 肩部角度 | **不动**，保留原姿势 | 强制设为 36° 下垂 |
| 肘部 | 拉直 | 也会拉直（已内置同款修正） |
| 腕部 | 拉直 | 不处理 |
| 适用 | 只想消除手臂预弯 | T-Pose → A-Pose 整体转换 |

> 注：`convert_to_apose` 同时修复了一个历史 bug —— 旧代码 `bone.select = True`（`PoseBone` 无 `select` 属性）在 Blender 3.x 会直接报错，已改为 `bone.bone.select = True`。
